# homepage_spec.md

## 1. 문서 목적
이 문서는 TripNode의 홈페이지(랜딩 페이지) UX/UI 방향, 정보 구조, 핵심 메시지, 시각 연출 원칙을 정의하기 위한 문서이다.

TripNode 홈페이지는 단순 소개 페이지가 아니라, 사용자가 서비스의 핵심 가치인  
**“지도 위에서 직접 여행 노드를 배치하고, 배치된 노드를 기반으로 사용자의 취향/ 여행스타일을 반영한 경로를 제공받는 경험”**  
을 첫인상에서 직관적으로 느끼게 만드는 것을 목표로 한다.

---

## 2. 홈페이지의 핵심 목표

TripNode 홈페이지는 아래 3가지를 빠르게 전달해야 한다.

1. **TripNode가 무엇인지**
   - 국내 여행자를 위한 지도 기반 맞춤형 여행 일정 생성 서비스

2. **기존 여행 서비스와 무엇이 다른지**
   - 추천만 받는 것이 아니라, 사용자가 직접 지도 위에 여행 노드를 놓아 여행계획을 짜는 사용자 참여형 경험

3. **왜 써보고 싶어지는지**
   - 복잡한 여행 관련 서치 과정 없이도 빠르고 직관적으로 나만의 여행 코스를 설계할 수 있음.

---

## 3. 핵심 UX 철학

### 3.1 시각 경험 우선
TripNode는 텍스트 설명보다 시각적 이해가 먼저 일어나야 하는 서비스이다.  
따라서 홈페이지는 긴 설명문 중심이 아니라, **지도 / 노드 / 연결 경로 / 배치 인터랙션** 이 즉시 보이는 구조여야 한다.

핵심 원칙:
- 텍스트만으로 설명하지 않는다.
- 지도 위에 노드가 놓이고 경로가 연결된 장면을 반드시 함께 보여준다.
- 사용자는 “읽어서 이해”하는 것이 아니라 “보자마자 감을 잡아야” 한다.

### 3.2 차별점은 첫 화면에서 드러나야 함
TripNode가 단순한 여행 추천 서비스나 블로그형 여행 정보 서비스처럼 보이면 안 된다.

첫 화면에서 반드시 아래 요소가 보여야 한다.
- 지도
- 여러 개의 여행 노드
- 노드 간 연결된 경로
- 사용자가 직접 설계하는 듯한 느낌

### 3.3 사용자 참여형 설계 경험 강조
TripNode의 핵심은 사용자의 취향과 노드를 놓음으로써 설계되는 여행 경로를 기반으로 
여행 코스를 추천받는 경험이다.
기존까지는 리뷰도 직접 알아보고, 오픈시간, 닫는 시간 이런 거를 각각 사이트를 들어가봐서 알 수 있었는데
이제는 한 앱에서 그런거를 관리해주는 것.
홈페이지에서도 “내가 직접 설계하는 재미와 직관성”을 우선적으로 보여준다.

## 4. 핵심 브랜드 메시지

### 메인 카피
**혼자 떠나는 여행, 지도 위에서 직접 설계하세요**

이 문구는 TripNode 랜딩 페이지의 대표 메시지로 활용한다.

### 서브 카피 방향
- 원하는 장소를 노드처럼 배치하고
- 동선을 비교하며
- 혼자만의 여행 코스를 빠르게 완성할 수 있다는 점을 짧고 명확하게 전달한다

예시 톤:
- 가고 싶은 장소를 지도 위에 직접 놓아보세요.
- 경로를 눈으로 비교하고, 나만의 여행을 더 직관적으로 설계하세요.

### Section 1. Hero

#### 목적
서비스의 정체성과 차별점을 첫 화면에서 즉시 전달한다.

#### 반드시 들어가야 할 요소
- 메인 카피
- 짧은 서브 카피
- CTA 버튼
- 지도 기반 핵심 비주얼

#### 메인 카피
**혼자 떠나는 여행, 지도 위에서 직접 설계하세요**

#### 서브 카피 방향
- 여행할 때 복잡한 계획을 쉽게 만들 수 있다는 메시지
- 노드 배치 / 경로 비교 / 빠른 설계 경험 강조

#### CTA 예시
- 여행 코스 바로 만들기
- 서비스 미리보기

#### 비주얼 연출 원칙
Hero 영역에서는 텍스트만 있으면 안 된다.  
반드시 다음 장면이 함께 보여야 한다.

- 지도 위에 여러 개의 여행 노드가 놓여 있음
- 노드들이 경로로 연결되어 있음
- 사용자가 직접 배치/수정하는 듯한 느낌이 남
- 정적인 소개 이미지보다, 약간의 움직임이 있는 인터랙션형 연출이 더 적합함

#### 비주얼의 역할
Hero의 비주얼은 단순 장식이 아니라, TripNode의 차별점을 한 번에 이해시키는 핵심 설명 장치이다.

### Section 2. 문제 공감

#### 목적
기존 여행 계획 방식의 번거로움을 공감시키고, TripNode가 그 불편함을 어떻게 줄이는지 대비시킨다.

#### 메시지 방향
기존 여행 준비는 아래와 같은 불편이 있다.
- 지도 앱에서 길찾기 따로 봐야 함
- 리뷰와 장소 정보를 따로 확인해야 함
- 저장한 장소들을 머릿속으로 비교해야 함
- 전체 동선을 직관적으로 보기 어려움

#### 추천 레이아웃
**좌측: 기존 방식의 불편함**  
**우측: TripNode 방식의 직관성**

#### 좌측 화면 연출
왼쪽에는 기존 여행 준비 방식의 복잡함을 시각적으로 보여준다.

예시:
- 네이버 지도 길찾기 화면
- 장소 리뷰 화면
- 검색 결과 화면
- 저장 목록 같은 화면

이 화면들을 정갈하게 하나만 보여주기보다,  
**여러 창이나 카드가 겹쳐진 듯한 방식으로 배치** 하여  
“여러 앱/정보를 왔다 갔다 봐야 하는 피로감”을 표현한다.

#### 우측 화면 연출
오른쪽에는 TripNode의 방식이 더 직관적이고 단순하다는 점을 보여준다.

예시:
- 지도 위에 여행 노드가 배치되어 있음
- 노드 간 경로가 연결되어 있음
- 전체 루트가 한눈에 들어옴
- 시각적으로 입체감 있는 3D 느낌의 연출 가능

여기서는 단순 지도 스크린샷보다  
**TripNode만의 제품 경험처럼 보이는 장면** 을 만들어야 한다.

#### 섹션 핵심 의도
이 구간은 “불편함 설명”이 아니라  
**기존 방식 vs TripNode 방식의 체감 차이** 를 시각적으로 납득시키는 구간이다.

### Section 3. 핵심 차별점

#### 목적
TripNode의 본질적인 강점을 짧고 명확하게 정리한다.

#### 핵심 차별점 3가지
1. 지도 위 직접 배치
2. 노드 이동 / 비교 중심 설계
3. 혼자 여행에 맞는 빠른 코스 완성

#### 카드 예시 방향
- **지도 위에 바로 놓기**
  - 가고 싶은 장소를 노드처럼 추가하고 배치할 수 있다.

- **경로를 눈으로 비교하기**
  - 어느 루트가 더 자연스러운지 직관적으로 확인할 수 있다.

- **혼자 여행에 맞춘 빠른 설계**
  - 당일치기부터 짧은 여행까지 부담 없이 빠르게 계획할 수 있다.

#### UX 원칙
이 섹션은 텍스트를 길게 쓰지 말고,  
짧은 문장 + 아이콘/일러스트/미니 UI 조각으로 구성한다.

---

### Section 4. 사용 흐름

#### 목적
TripNode를 어떻게 사용하는지 쉽고 빠르게 이해시키는 구간이다.

#### 추천 단계
1. 장소 검색
2. 노드 추가
3. 순서/위치 조정
4. 경로 확인 및 일정 저장

#### 표현 방식
단순 텍스트 나열보다 아래 방식이 좋다.
- step 카드
- 간단한 UI 스냅샷
- 흐름형 다이어그램

#### 전달 포인트
사용자는 “생각보다 쉽다”는 인상을 받아야 한다.  
복잡한 기능 설명이 아니라,  
몇 단계만 거치면 여행 코스를 만들 수 있다는 느낌이 중요하다.

### Section 5. 예시 코스 / 결과물

#### 목적
TripNode를 사용하면 실제로 어떤 결과가 만들어지는지 보여준다.

#### 예시 방향
- 서울 당일치기 혼행 코스
- 부산 감성 혼행 코스
- 전주 먹거리 중심 코스

#### 표현 방식
- 실제 지도 위 경로 예시
- 노드가 배치된 플랜 카드
- 간단한 일정 요약 카드

#### UX 목적
사용자가  
“아, 나도 이런 식으로 금방 만들 수 있겠네”  
라고 상상하게 만들어야 한다.

### Section 6. 브랜드 철학

#### 목적
TripNode가 왜 이런 방식으로 만들어졌는지 브랜드 정체성을 전달한다.

#### 메시지 방향
TripNode는 자동 추천만 하는 서비스가 아니다.  
사용자가 지도 위에서 직접 생각하고, 비교하고, 선택하는 과정을 더 즐겁고 직관적으로 만들기 위한 서비스이다.

#### 작성 원칙
- 길지 않게
- 선언형 문장 중심
- 브랜드 태도를 보여주되 과하게 추상적이지 않게

예시 문장:
TripNode는 여행 계획을 대신 결정해주는 서비스가 아니라,  
직접 설계하는 과정을 더 즐겁고 직관적으로 만들어주는 서비스입니다.

### Section 7. 최종 CTA

#### 목적
홈페이지를 본 사용자가 실제 행동으로 이어지도록 유도한다.

#### 추천 CTA
- 지금 여행 코스 만들어보기
- 혼자 여행 설계 시작하기

#### 보조 문구 방향
- 부담 없이 시작 가능
- 빠르게 체험 가능
- 저장/로그인은 이후 단계로 유도 가능

## 7. 디자인 톤 & 무드

### 전체 방향
- 깔끔하고 직관적이어야 함
- “여행 서비스”의 감성은 살리되, 지나치게 감성 사진 위주가 되면 안 됨
- 제품 UI 자체가 가장 중요한 비주얼 자산이어야 함

### 무드 키워드
- 지도 중심
- 노드 중심
- 경로 가시성
- 직관적
- 가볍지만 정교한 느낌
- 사용자 참여형
- 설계하는 재미

### 피해야 할 방향
- 일반 여행 블로그/매거진 같은 분위기
- 감성 사진만 많은 구조
- AI 추천 서비스처럼 보이는 과장된 톤
- 텍스트 설명이 지나치게 많은 랜딩 페이지

## 8. MVP와의 정합성

현재 MVP 우선순위는 다음과 같다.

1. 지도 표시
2. 장소 검색
3. 노드 추가 / 이동 / 삭제
4. 경로 계산
5. 일정 저장
6. LLM 추천

따라서 홈페이지도 이 우선순위를 반영해야 한다.

### 원칙
- 홈페이지 전면에는 지도 기반 설계 경험을 배치한다.
- 아직 LLM 추천은 보조 요소로만 다룬다.
- TripNode의 첫 인상은 “AI 추천 앱”이 아니라 “지도 기반 여행 설계 툴”이어야 한다.

---

## 9. 구현 시 반드시 유지해야 할 원칙

### 필수 원칙
- 첫 화면에서 지도 + 노드 + 경로가 보여야 한다.
- 텍스트보다 시각 경험이 먼저 전달되어야 한다.
- 문제 공감 섹션에서는 기존 방식과 TripNode 방식을 강하게 대비시켜야 한다.
- TripNode는 사용자가 직접 설계하는 경험이 핵심임을 유지해야 한다.
- CTA 문구는 추상적이지 않고 구체적인 행동 중심이어야 한다.
